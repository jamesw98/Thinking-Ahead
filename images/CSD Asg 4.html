Assignment 4
13%, due before 11:00 p.m. on November 5, 2019

Submission

Part 1:

  1. Name your cloud software application as Videos-YourLastName in NetBeans, with first letter of your last name capitalized.
  2. Document all XHTML and Java code to demonstrate your knowledge similar to the tutorials.
  3. The top documentation in an XHTML and Java class file must have your full name as the creator of the code.
  4. Right click your NetBeans project folder and select Compress to create a zip file.
  5. Rename the zip file created as Yourlastname-Asg4.zip with first letter of your last name capitalized.
  6. Upload your zip file to your account in Google Drive, right click the uploaded file and select Share from the pop-up menu, enter balci@vt.edu, add a note as CS3754 Asg 4 Submission - yourFullName, and click Send.

Part 2:

  1. Deploy your cloud software application to:
    • Jupiter if your last name starts with letters A through K or to
    • Venus if your last name starts with letters L through Z.

  2. Your cloud software application must be usable before the deadline by entering the following URL on a desktop, laptop, tablet, or smartphone computer:
    • https://jupiter.cs.vt.edu/Videos-YourLastName, or
    • https://venus.cs.vt.edu/Videos-YourLastName.

Submission Policy:

Both Part 1 and 2 must be completed before 11:00 p.m. on the due date.

There will be a 10% penalty per day late. Lateness is determined with respect to the time of your completion of Part 1 and 2 above. For example, an assignment due on Tuesday: if submitted (Part 1) and deployed (Part 2) on Wednesday before 11:00 p.m. is considered one day late; on Thursday before 11:00 p.m. is considered two days late; etc. No assignment will be accepted if late more than three days. Your assignment Part 1 and 2 must be complete when submitted. Partial submissions are not allowed.

Public Videos

This mode of operation lists favorite YouTube videos selected by the public. Anyone can perform CRUD operations on this list. Selecting Public Videos from the menu in the header shall display the following.

All of the data shown above (except thumbnail image) shall be retrieved from the UsersVideosDB database (UsersVideosDB.sql is provided).

The title of the data table shall be Public List of Favorite YouTube Videos

The Id column:

  • width = 20 px
  • align = center

The Title column:

  • sortable
  • width = 100 px
  • align = default (left)

The Thumbnail column:

  • width = 80 px
  • align = center
  • Obtain the thumbnail video image from
    https://i1.ytimg.com/vi/#{item.youtubeVideoId}/hqdefault.jpg
  • Style the image with class="scalableImage" in the CSS file.

The Description column:

  • width = 250 px
  • align = default (left)

The Duration column:

  • sortable
  • width = 80 px
  • align = center

The Date Published column:

  • sortable
  • width = 100 px
  • align = center

The Category column:

  • sortable
  • width = 80 px
  • align = center

The View, Edit, Delete, Play, and Share buttons shall be inactive unless a data table row is selected.

Clicking the Create button in the footer facet of the data table shall display a dialog box with the Create (hidden) and Cancel buttons for the user to enter the values required to create a new YouTube public favorite video as shown below.

Clicking the Create button in the dialog box shall create the new public video in the UsersVideosDB database, display the message "Public Video was successfully created." as shown below, and list it in the data table immediately.

The newly created video must be listed in the data table immediately as shown below.

VIEW

After selecting a video in the Public List, clicking the View button shall display a dialog showing the uneditable attributes of the selected video as shown below.

EDIT

After selecting a video in the Public List, clicking the Edit button shall display a dialog showing the attributes of the selected video as editable as shown below.

The user shall be able to change the data and click the Save button to save the changes in the UsersVideosDB database and display the message "Public Video was successfully updated." as shown below.

DELETE

After selecting a video in the Public List, clicking the Delete button shall display a confirmation dialog as shown below.

Clicking the Yes button in the dialog shall (a) display the "Public Video was successfully deleted." message shown below, (b) delete the video from the UsersVideosDB database, and (c) immediately update the data table.

PLAY

After selecting a video in the Public List, clicking the Play button shall display the Play Video dialog with the Close button as shown below. Clicking the Play icon button in the Play Video dialog shall play the YouTube video as embedded within the dialog as shown below. The video player shall display all of the controls.

SHARE

After selecting a video in the Public List, clicking the Share button shall display a confirmation dialog as shown below.

Clicking the Yes button in the dialog shall display the "Unable to Share!" message below if there is no user signed in.

Clicking the Yes button in the dialog shall add the selected video to the signed-in user's list of favorite YouTube videos and display the message "User Video was successfully created." as shown below.

The shared video shall be displayed in the user's favorite list immediately.

SEARCH

Clicking the Search button shall display the Search Public Videos dialog with the Search and Cancel (hidden) buttons as shown below.

The Search capability shall be implemented to search for the Search String entered by the user in the following Search Fields:

  • Video Title containing the Search String entered by the user.
  • Video Description containing the Search String entered by the user.
  • Video Category containing the Search String entered by the user.
  • All = Video Title, Video Description, OR Video Category containing the Search String entered by the user.
